1688 智能选品匹配 — 功能方案说明

本文档面向客户沟通使用，说明「卖家精灵表格 → 自动匹配 1688 供应商 → 利润核算」功能的实现方案。

一、客户需求

客户从卖家精灵下载"待开发产品"Excel 表格，包含亚马逊商品的图片、英文标题、售价、重量、FBA 费用等信息。希望系统能够：

  1. 自动解析上传的表格
  2. 自动在 1688 上搜索最匹配的供应商商品，拿到采购价
  3. 自动核算利润，判断该产品是否值得开发

目标：全自动，上传表格 → 输出利润分析报告，中间无需手动操作。

以客户提供的样本 Product-Home&Kitchen-US-Last-30-days-419484.xlsx 为例：

三、全自动化流程

用户上传卖家精灵 Excel（最多 100 条）
        ↓
   ① 解析表格 — 校验必填字段（商品标题、售价、商品主图）
        ↓
   ② 确认参数 — 头程运费（预设/自定义）、国内运费（默认0）、汇率（实时/手动）、佣金比例（按行大类目，缺则15%）
        ↓
   ③ 创建异步分析任务（页面不卡住，显示进度面板）
        ↓
   对每条商品逐条处理（实时更新进度）：
   ├─ ③a AI 智能翻译（Gemini）
   │      英文标题 → 中文采购关键词
   │      例："Disposable Toilet Brush with 48 Refills"
   │         → "一次性马桶刷 替换头 48个装"
   ├─ ③b 1688 商品搜索（DajiAPI）
   │      ├─ 通道一：中文关键词搜索（已实现）
   │      └─ 通道二：产品主图以图搜图（待扩展，Daji 以图搜图仅支持 1688 域名图片）
   │      → 当前仅用关键词通道，返回候选结果
   ├─ ③c AI 智能匹配打分（Gemini）
   │      对比原始产品 vs 1688 搜索结果
   │      综合：标题相似度、规格参数、图片相似度、店铺评分
   │      → 选出 Top 3，默认推荐第 1 名
   │      → 搜不到则标记「未找到」，不扣积分
   └─ ③d 利润自动核算
          亚马逊售价（表格已有）
          - 1688 采购价（API 获取，用推荐第 1 名的价格）
          - FBA 费用（表格已有 / 按重量估算）
          - 头程运费（用户选定的费率 × 重量）
          - 平台佣金（售价 × 品类佣金率）
        ↓
   ④ 查看完整利润分析报告
      → 默认按毛利率排序
      → 每行可切换 Top 3 匹配，利润实时重算
      → 「保存报告」→ 存入历史记录
      → 「导出 Excel」→ 下载完整报告

四、利润计算公式

单件毛利 = 亚马逊售价 × 汇率
         - 1688 采购价
         - 国内运费（默认 0，用户可设置每件 ¥X）
         - 头程运费（用户选定费率 × 重量）
         - FBA 配送费（表格已有 / 缺失时按重量估算）
         - 亚马逊佣金（售价 × 品类佣金率，缺类目用 15%）
         - 其他费用（预留：广告、退货，暂不参与）

毛利率 = 单件毛利 ÷ 亚马逊售价 × 100%

各参数默认值（已确认）：

  • 汇率：默认实时获取（Frankfurter API，免费无需 Key），用户可切换为手动输入固定值
  • 头程运费：双模式
    • 预设费率（默认选中空运）：空运 ¥40/kg、海运快船 ¥12/kg、海运慢船 ¥8/kg
    • 自定义：用户手动输入 ¥/kg 单价
  • 国内运费：默认 0，用户可设置「每件 ¥X」（采购到仓库的国内段费用）
  • 品类佣金率：按行 大类目 映射 Amazon 官方 Referral Fee，混品类表格每行独立计算；缺类目时用默认 15%
  • 广告费 / 退货率：预留，暂不参与计算（界面显示「即将支持」）

1688 数据获取方案（核心）

由于 1688 没有免费公开的搜索 API，需要通过以下方式之一获取数据：

方案 优点 缺点 推荐度
第三方 1688 API 服务商 接入简单、数据稳定、合规 按次收费 ⭐⭐⭐ 首选
1688 开放平台官方 API 最正规、数据最全 需企业资质、审核周期长 ⭐⭐ 长期
网页爬虫 免费 反爬强、不稳定、有法律风险 ⭐ 不推荐

推荐方案：先接入第三方 1688 API 服务商（国内有多家提供此类服务），快速上线；后续如有需要再申请官方 API。

方案 E：1688 官方开放平台（长期方案）

  • 必须企业资质（个人开发者无法申请核心 API）

  1. 在 open.1688.com 注册开发者账号
  2. 企业实名认证（1~2 个工作日）

  • 调用限制：个人 ≤10 次/秒，企业 ≤50 次/秒

优点：

  • 数据最全、最准、最稳定
  • 官方以图搜图效果最好
  • 长期使用最合规

缺点：

  • 需企业资质，申请周期 1~2 周
  • 部分 API 权限受限

用户操作流程

步骤 1：上传表格
  → 拖拽或点击上传卖家精灵 Excel（最多 100 条产品）
  → 系统自动识别列名，校验必填字段，预览数据

步骤 2：确认设置
  → 头程运费：选择物流方式（空运/海运快船/海运慢船）或手动输入 ¥/kg
     默认：空运 ¥40/kg
  → 国内运费：默认 0，用户可设置「每件 ¥X」（采购到仓库）
  → 汇率：默认「实时汇率」（Frankfurter API 自动获取），可切换为手动输入固定值
  → 佣金比例：按行大类目自动填充（混品类每行独立），缺类目用 15%，用户可手动调整
  → 广告费/退货率：预留（暂不参与计算，界面待补充）
  → 点击「开始分析」

步骤 3：分析进行中（异步任务，页面不卡住）
  → 生成一个分析任务，显示任务进度面板
  → 进度条 + 文字提示（如「正在分析第 12/46 条…」）
  → 每完成一条，实时在表格中更新该行结果
  → 用户可以看到已完成的行、正在处理的行、等待中的行
  → 页面可自由浏览，不阻塞其他操作

步骤 4：查看报告
  → 完整利润分析表格
  → 每个产品展示：
     - AI 推荐的最佳 1688 匹配商品（默认选中，标记「推荐」）
     - 另外 2 个备选匹配（可点击切换，利润实时重算）
     - 匹配不到的标记「❌ 未找到」，利润列显示「—」
  → 支持按毛利率排序，一眼看出哪些产品值得开发
  → 「保存报告」— 存入历史记录，可随时回看
  → 「导出 Excel」— 下载完整报告

匹配结果逻辑

  • 每条产品通过关键词搜索在 1688 上查找（以图搜图待扩展）
  • AI（Gemini）对搜索结果进行智能打分，综合标题相似度、规格参数、图片相似度、店铺评分
  • 返回 Top 3 匹配结果，默认选中评分最高的第 1 名用于利润计算
  • 用户可手动切换到第 2、3 名，利润自动重算
  • 搜不到匹配商品的行标记为「❌ 未找到」，不扣积分
