侵权风险检测 - 外部数据源集成可行性说明

本文档整理自客户咨询：是否可将美国版权局与专利汇的查询能力集成到 PicToolAI 侵权风险检测中。供决策参考。

问题：能否将这两个网站的功能集成到 PicToolAI 程序中？

客户常使用的两个外部站点：

  1. publicrecords.copyright.gov — 美国版权局公开记录检索
  2. patenthub.cn — 专利汇（专利检索与数据分析）

以下分别说明集成的可行性与工作量。

一、专利汇（patenthub.cn）— 可以集成

结论

可行。专利汇提供完整 REST API，可按需调用，技术上容易对接。

说明

  • 有标准 REST API，支持专利搜索、详情、相似专利、法律状态等
  • 需申请 TOKEN：访问 patenthub.cn/api/open，商务 QQ：2157717237
  • 按调用量计费，具体价格需与对方洽谈

与现有侵权检测的对比

维度 现有方案（SerpApi + Google Patents） 专利汇 API
覆盖范围 美国专利为主 中国专利 + 全球专利
接入方式 第三方搜索引擎抓取 官方 API 直连
成本 SerpApi 约 $75/月（约 5000 次） 需与专利汇洽谈
适用场景 美国市场侵权风险 中国市场、全球专利

集成方式

  • 在侵权风险检测「深度查询」流程中，用 AI 提取产品/外观关键词调用专利汇 /api/s 进行专利检索
  • 将检索结果交给 Gemini 做风险分析与报告整合

已实现（2026-03）

  • 后端 server/index.js 深度查询流程已集成专利汇 /api/s 接口
  • 环境变量：PATENTHUB_TOKEN（在 server/.env 中配置，不配置则跳过）
  • 与 Google Patents 并行调用，结果合并后一并交给 Gemini 综合报告
  • 专利汇检索范围：ds=all（中国+全球），取前 5 条
  • 报告「本次检索方式」中会显示「Google Patents + 专利汇」当两者均配置时

二、美国版权局（publicrecords.copyright.gov）— 难度较大

结论

不提供实时 API。若要实现类似功能，需自建数据索引，开发与运维成本较高。

说明

  • 无公开实时查询 API，版权局仅提供网页检索
  • 提供批量数据下载：1978–2025 年约 2,200 万条记录（MARC、CSV 等），可在 copyright.gov/economic-research/usco-datasets 下载
  • 实时查询依赖官网界面或付费研究服务

若自建版权检索

  1. 下载并解析版权局批量数据（规模数十 GB 级）
  2. 建立检索索引（如 Elasticsearch），支持关键词、权利人、作品名等检索
  3. 定期（如每季度）同步更新数据
  4. 准备存储与计算资源

工作量评估

  • 开发：大（数据下载、解析、索引构建、查询接口、定期更新）
  • 运维：需持续存储与计算资源，定期同步数据

替代方案

  • 短期：在侵权检测页面增加「美国版权局登记查询」入口，提供官网链接与使用说明，引导用户到官网手动查询
  • 中长期：若客户确有高需求，再评估是否自建索引

综合建议

方案 可行性 工作量 建议
集成专利汇 ✅ 可行 中等 建议推进，尤其有中国/全球专利需求时
集成版权局检索 ⚠️ 复杂 大 可先提供官网跳转，自建检索待评估

建议步骤

  1. 专利汇：联系专利汇获取 API TOKEN 与报价，评估调用量、成本与 SerpApi 的对比
  2. 版权：先增加「美国版权局官网」查询入口与指引，满足基本需求；自建检索作为后续可选规划
  3. 商标：继续使用现有 SerpApi（Google 搜索/USPTO），如需更专业商标比对，可另做方案评估

附录：专利汇主要 API 接口

接口 用途
/api/s 专利搜索（ds=cn 中国 / ds=all 全球）
/api/patent/base 基本信息
/api/patent/detail 详情
/api/patent/like 相似专利
/api/patent/citing 引用数据
/api/patent/tx 法律事务
/api/ls 批量法律状态
